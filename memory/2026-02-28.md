## Key Files

| File | Purpose |
|------|---------|
| `.github/workflows/youtube-uploads.yml` | Workflow definition (cron + steps) |
| `upload_scheduled.py` | Main upload logic (Drive → YouTube) |
| `authenticate_youtube.py` | One-time OAuth setup script |
| `processed_videos.json` | Tracks uploaded video IDs (prevents duplicates) |
| `upload_history.json` | Full upload history with timestamps |
| `daily_upload_count.json` | Tracks uploads per day |
| `token.pickle` | Refreshed OAuth token (committed by workflow) |

---

## Upload Flow

1. **Workflow triggers** (cron or manual)
2. **Decodes secrets** → creates `client_secret.json`, `service-account-key.json`
3. **Runs `upload_scheduled.py`**:
   - Loads `token.pickle` for YouTube API
   - Loads service account for Drive API
   - Lists videos in Drive folder
   - Filters out already-processed videos
   - Picks random unprocessed video
   - Downloads to `/tmp`
   - Uploads to YouTube with random title
   - Marks video as processed
   - Updates tracking files
4. **Workflow commits** refreshed `token.pickle` + tracking files back to repo

Random selection from `TITLES` array + sequential number:
- Emotional/extreme: "Can ANY Car Survive THIS Impact? 💀"
- Curiosity: "You Won't Believe What Happened to This Car!"
- Superlative: "The CRAZIEST Crash Tests of 2026!"
- Numbered: "10 Cars That Didn't Stand a Chance!"

Format: `{random_title} #{upload_count}`

To build another automation exactly like this:
