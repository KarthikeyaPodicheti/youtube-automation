## Self-Improvement Workflow

When errors, corrections, or missing capabilities occur in this workspace, follow this workflow:

1. Decide which file to log in:
   - `.learnings/ERRORS.md` – for failing commands, stack traces, or repeated operational problems.
   - `.learnings/LEARNINGS.md` – for durable lessons, best practices, conventions, or fixes that might help in the future.
   - `.learnings/FEATURE_REQUESTS.md` – for missing tools or capabilities that would significantly improve productivity.
2. Add a new entry using the self-improving-agent format, including:
   - A unique ID (e.g., LRN-YYYYMMDD-001, ERR-YYYYMMDD-001, FEAT-YYYYMMDD-001).
   - Date, area / context, short title, detailed description, and proposed next actions.
3. Regularly review these files and promote broadly useful insights into:
   - `MEMORY.md` / `CLAUDE.md` – project facts, domain knowledge, and conventions.
   - `AGENTS.md` – workflows, automation patterns, and collaboration rules between agents.
   - `SOUL.md` / `TOOLS.md` – behavioral guidelines, tool gotchas, and safety constraints.

As the assistant, whenever you encounter a repeated error, user correction, or obvious improvement opportunity, propose a concrete entry for the relevant `.learnings/*.md` file and, when appropriate, suggest promotions to the long-term memory files listed above.
